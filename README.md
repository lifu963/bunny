# bunny

不调用外部库，手搓一个完整的后端项目（造轮子）。

完整的后端项目应包括（个人理解）：

1. 获取外部数据的能力
2. 对内部数据增删改查的能力

因此，本项目的架构设想如下：

![Architecture](https://github.com/lifu963/bunny/assets/56394323/2d871230-435d-4bfd-a872-bb15c9b53900)

当然目前我还没有过完整的项目经验，这一套没准挺扯淡的。
